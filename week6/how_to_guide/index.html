Netflix Roulette Api

How to guide

By: Kevin Turkington

CS 290


Purpose:

This API is the replacement for the official netflix search API. In late 2014 the Official Netflix public API was shutdown. Despite the name This Netflix API is NOT a random movie generator but similar to a movie search engine that works via get requests.

Offcial Netflix Roulette API documentation can be found here

Misc:

Keep in mind this API is a non official tool that can be used by the public without any verification keys. keep in mind usage and bandwidth of requests.


How to make an API request?

Requests can be made via Get requests to the following url:

http://netflixroulette.net/api/api.php?

Parameters for GET requests:

  • Genre (string)
  • Movies (boolean)
  • TV (boolean)
  • LowRating (float 1-5)
  • HighRating (float 1-5)
  • Director (string)
  • Keyword (string)
  • Year (int)
  • type (string)

Sample code: (JavaScript)

var API_URL = http://netflixroulette.net/api/api.php?; //note the question mark at the end

var title_value = "ip man";
var predicate = "title=" + URLEncodeComponent(title_value);//making string readable for get requests

var searchURL = API_URL + predicate;

console.log(searchURL);
// output should be: http://netflixroulette.net/api/api.php?title=ip%20man

description: display movie data from title alone.
Disred output: http://netflixroulette.net/api/api.php?title=ip%20man


Parameter notes:


Genre

This variable is very straight forward and will match directly with genres in the actual netflix web application. However in some cases genres such as action will need "movies" appendend to the end.

For example:

var genre = "?genre=action";

genre = genre + " movies";
console.log(genre);//console should display ?genre=action movies

var predicate = URLEncodeComponent(genre);
console.log(predicate);// console should display ?genre=action%20movies


Movies

This variable is helpful for the sorting movies for the API. It is a simple True/False if the user wants to see movies or not. The default for this function is True.


TV

This variable is similar to movies however it is used for sorting Tv shows in its search. The default is False.


LowRating

This Variable helps sort out any movies below the threshold. The lowest this variable can be is 1.0, and can be a float (1.3,2.3,4.9). This highest this variable can be set is 5.0. The default is 1.0.


HighRating

This Variable helps sort out any movies above this threshold. The lowest this variable can be is 1.0, and can be a float like LowRating. The highest value this variale can be set is 5.0. The default is 5.0.


Director

This variale will return any movies with a director name that is similar to the one given. Full and partial names can be used such as James, or James Franco.


Keyword

Any movies with descriptions that match the keyword will be returned. This search criteria is not the most effective for finding movies you may want.


Year

Any movies made during a specific year will be returned.Any movies or tv shows that were made during a different year will not be returned, even if exact movie titles or director names are used.


Type

This specifys the data type sent by the API to the client. For example JSON, XML, as well as other formats can be returned to the client to be processed.

    Full list:
  • JSON
  • XML
  • plain


How to use:

The data recieve can vary from parameter broadness. But the API seems to have a return movie threshold of roughly 1000 movies. Any searches that are too broad will return a 400:bad request error. Additionally Any searches too specific with bad information will return a 404:Not found error.
